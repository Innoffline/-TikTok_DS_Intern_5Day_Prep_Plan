统计学速记卡片 — 核心概念
必背清单：
• p-value：在H0为真时，观察到当前或更极端结果的概率。p < 0.05 → 拒绝H0
• Type I Error (α)：假阳性，H0为真却拒绝 → 对应 Precision
• Type II Error (β)：假阴性，H0为假却未拒绝 → 对应 Recall
• t-test vs z-test：样本小(<30)或方差未知用t-test；大样本+方差已知用z-test
• 置信区间 CI：sample mean ± Z * SE，其中 SE = σ/√n
• Central Limit Theorem：n足够大时，样本均值近似正态分布
60min
Precision vs Recall 深入理解
Trust & Safety 核心考点！
• Precision = TP/(TP+FP)：预测为正中实际为正的比例
• Recall = TP/(TP+FN)：实际为正中被正确预测的比例
• T&S场景：
  - 封禁账号：优先Precision（误封好用户代价极高）
  - 检测儿童安全风险内容：优先Recall（漏检代价极高）
  - Spam检测：通常需要平衡，但偏Precision

下午 (4h) — A/B Testing 系统复习
A/B Testing 核心概念速记
面经最高频考点之一！
• A/B的本质：通过随机分组建立因果关系（不是相关关系）
• 不做随机分配的后果：confounding variables → selection bias → 无法归因
• 需要causal inference的场景：无法随机分配时(如网速对使用率的影响)
• MDE (Minimum Detectable Effect)：最小可检测效果
  - MDE = f(α, β, n, σ²)：样本越大MDE越小
  - 100个样本分很多组 → 每组样本小 → MDE变大 → 更难检测到效果
• 几百人的小群体怎么跑实验：增加实验时长、用更sensitive的指标、减少分组数
60min
A/B Test 结果解读 — 面经场景题
必须能回答的场景：
• 实验跑了7天，核心指标没有显著变化 → 是否继续？
  答：检查power是否足够、检查novelty effect、延长实验、看secondary metrics
• A组GMV提升但转化率下降 → 如何解读？
  答：可能客单价提高（高价商品），用户数减少但花费更多，Simpson's Paradox
• 老用户GMV上升+新用户GMV下降 → 策略健康吗？
  答：不健康，说明获客能力下降，长期增长受限
60min
Causal Inference 核心方法速记
• DiD (Difference-in-Differences)：假设条件=Parallel Trends；不满足 → 加controls或用Synthetic Control
• Propensity Score Matching (PSM)：无法做AB test时的替代方案
• Instrumental Variable：解决endogeneity
• Inverse Probability Weighting：对observational data做加权
• Stratification + ATE：加入所有confounder分层计算平均处理效应
45min
Sample Size Calculation 公式速记
n = (Z_α/2 + Z_β)² × 2σ² / δ²
其中 δ = MDE, σ² = 方差
chi-squared test：用于比较两组categorical variable（如conversion rate A vs B）

SQL 万能模板

1. 分组Top N (面经超高频)

WITH ranked AS ( SELECT *, RANK() OVER (PARTITION BY department_id ORDER BY salary DESC) as rk FROM employees ) SELECT * FROM ranked WHERE rk <= N;

2. 互相关注用户对 (面经原题)

SELECT DISTINCT LEAST(a.from_user_id, a.to_user_id) AS user1, GREATEST(a.from_user_id, a.to_user_id) AS user2 FROM follows a INNER JOIN follows b ON a.from_user_id = b.to_user_id AND a.to_user_id = b.from_user_id;

3. 过去30天Top 5互动用户 (面经原题)

SELECT from_user_id, COUNT(DISTINCT to_user_id) AS unique_interactions, COUNT(*) AS total_interactions FROM interactions WHERE action_date >= CURRENT_DATE - INTERVAL 30 DAY GROUP BY from_user_id ORDER BY unique_interactions DESC, total_interactions DESC LIMIT 5;

4. 连续N天登录 (差值法)

WITH numbered AS ( SELECT user_id, login_date, ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY login_date) AS rn FROM (SELECT DISTINCT user_id, login_date FROM logins) ) SELECT user_id, DATE_SUB(login_date, INTERVAL rn DAY) AS grp, COUNT(*) AS consecutive_days FROM numbered GROUP BY user_id, grp HAVING COUNT(*) >= N;

5. 留存率计算 (面经原题)

SELECT a.date, COUNT(DISTINCT b.advertiser_id) * 1.0 / COUNT(DISTINCT a.advertiser_id) AS retention_rate FROM ads a LEFT JOIN ads b ON a.advertiser_id = b.advertiser_id AND b.date = DATE_ADD(a.date, INTERVAL 1 DAY) GROUP BY a.date;

6. 转化率计算 (电商漏斗)

SELECT DATE(i.date) AS dt, COUNT(DISTINCT i.product_id) AS impressions, COUNT(DISTINCT c.product_id) AS clicks, COUNT(DISTINCT o.order_id) AS orders, COUNT(DISTINCT o.order_id) * 1.0 / NULLIF(COUNT(DISTINCT i.product_id), 0) AS conversion_rate FROM impressions i LEFT JOIN clicks c ON i.product_id = c.product_id AND i.date = c.date LEFT JOIN orders o ON c.product_id = o.product_id AND c.date = o.date AND o.status = 'paid' GROUP BY DATE(i.date);
